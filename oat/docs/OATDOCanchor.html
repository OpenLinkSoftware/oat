OAT.Anchor.assign(anchor, options)

Modifies anchor element so it acts like Enhanced Anchor: when activated, it displays some additional data in lookup bubble. The lookup bubble is an instance of OAT.Win object - so it can be styled the same way: use predefined OAT.Win styles or use its own via template. Options are following:

  • href - String, url of requested data. When false, anchor inherits this value from the href attribute.
  • connection - OAT.Connection object
  • datasource - OAT.DataSource object
  • result_control - "grid", "timeline" or "form". Default is grid. Use false for common window.
  • content - node to be inserted (for raw text can be used OAT.Dom.text() for example)
  • status - string to be inserted into status line
  • title - string to be in the title area of the window
  • activation - "hover", "click", "dblclick" or "focus"
  • width, height - dimensions of lookup window (false is like 'auto' in CSS)
  • elm - anchor node (use when other than current)
  • newHref - new 'href' attribute of the anchor, defaults to 'javascript:void(0)'
  • type - Window style type (default OST.WinData.RECT). See OAT.Win doc for other types.
  • template - custom type data - see OAT.Win doc for details.
  • visibleButtons - default "cr" (see OAT.Win)
  • enabledButtons - default "cr" (see OAT.Win)

var options = {
        title:"Title text",
        content:OAT.Dom.text("Content text"),
        status:"Status text",
        result_control:false,
        activation: "hover",
        type:OAT.WinData.TYPE_MAC
}
OAT.Anchor.assign("elementID", options);
